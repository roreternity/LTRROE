# LTRROE

**LTRROE** (Local Team Resource & Risk Optimization Engine) — исследовательский прототип
для стохастической оценки календарных рисков проекта. Объединяет календарное планирование
на графе (CPM), симуляцию Монте-Карло, генерацию синтетических данных и ML-базлайны, чтобы
сравнить *центральную* оценку срока с *правым хвостом* распределения сроков.

> 🇬🇧 English version: [README.md](README.md)

## Идея

Детерминированный CPM даёт одно число. Это число близко к **центральному сценарию**, а не к
безопасному дедлайну. LTRROE рассматривает срок проекта как распределение и оценивает:

- `det_duration_days` / `P50` — центральный исход (CPM ≈ P50, следует из ЦПТ);
- `P90` — осторожный исход;
- `Schedule Risk Ratio = (P90 − P50) / P50` — относительный размер буфера риска.

**Ключевой результат (синтетика, 10 000 проектов):** *срок* проекта предсказуем по
агрегированным признакам (R² ≈ 0.78), а *относительный резерв* — нет (R² ≈ 0.08). Этот
разрыв и есть «граница предсказуемости»: фиксированный процентный буфер необоснован, каждому
проекту нужен свой стохастический анализ.

## Структура

```text
ltrroe/
├── ltrroe/                 # пакет
│   ├── paths.py            # единый источник путей к данным/выходам
│   ├── core/               # языконезависимое ядро
│   │   ├── objects.py      # Project, Task, Employee, Dependency
│   │   ├── algorithms.py   # CPM (forward/backward), Монте-Карло
│   │   ├── visualisation.py
│   │   ├── test_data.py    # тестовый проект
│   │   └── demo.py         # запускаемое демо
│   └── synth/              # синтетические эксперименты
│       ├── project_level.py  # генерация датасета уровня проекта
│       ├── task_level.py     # генерация датасета уровня задач
│       ├── rf_project.py     # RF: срок + риск (уровень проекта)
│       ├── rf_task.py        # RF: длительность задачи
│       ├── xgb_model.py      # базлайн XGBoost
│       └── spearman.py       # чувствительность / корреляция
├── tests/                  # тесты pytest
├── outputs/                # сгенерированные CSV, модели, графики (в .gitignore)
├── data/                   # исходные данные, если есть (в .gitignore)
├── pyproject.toml
├── Makefile
└── README.md / README.ru.md
```

## Быстрый старт

```bash
make install     # pip install -e ".[dev,xgboost]"
make test        # запустить тесты
make demo        # демо на встроенном тестовом проекте
make all         # dataset -> ml -> figures
```

Или запустить этап напрямую:

```bash
python -m ltrroe.synth.project_level   # сгенерировать датасет уровня проекта
python -m ltrroe.synth.rf_project      # обучить Random Forest уровня проекта
```

Все артефакты попадают в `outputs/` (`outputs/files` — CSV/PKL, `outputs/figures` — PNG),
пути настраиваются в `ltrroe/paths.py`.

## Про датасет Gryzzly

Ранняя версия проверялась на публичном датасете Gryzzly. Он **исключён**: в нём нет полей,
от которых зависит модель (`task_skills` пуст — фактор навыков отключён, загрузка не
превышает порог). Текущий пайплайн — только синтетика; Gryzzly сохранён как
задокументированный неудавшийся эксперимент, а не как валидация.

## Зависимости

Python 3.9+, `numpy`, `pandas`, `matplotlib`, `seaborn`, `scipy`, `scikit-learn`, `joblib`
(опционально `xgboost`). Установка: `make install`.
